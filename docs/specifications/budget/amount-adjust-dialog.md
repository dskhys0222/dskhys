# 仕様：金額増減ダイアログ（Budget）

対象：`packages/client/budget` の `/` ルート

## 画面構成

- 現在金額表示
- 操作ボタン
  - `+`（増額）
  - `-`（減額）
- ダイアログ（モーダル）
  - タイトル：増額/減額（押下ボタンに応じて切替）
  - 入力：差分金額（数値）
  - ボタン：キャンセル / 確定

## 状態

- `currentAmount: number`
  - 初期値：0
- `dialogMode: 'increase' | 'decrease' | null`
  - ダイアログの表示モード（`null` のとき非表示）
- `deltaAmountInput: string`
  - 入力値（表示/入力用）

## 操作フロー

### 増額

1. ユーザーが `+` を押下
2. ダイアログを開く（モード：increase）
3. ユーザーが差分金額を入力
4. 確定押下でバリデーションを通過した場合、`currentAmount = currentAmount + delta`
5. ダイアログを閉じ、入力をリセットする

### 減額

1. ユーザーが `-` を押下
2. ダイアログを開く（モード：decrease）
3. ユーザーが差分金額を入力
4. 確定押下でバリデーションを通過した場合、`currentAmount = currentAmount - delta`
5. ダイアログを閉じ、入力をリセットする

### キャンセル

- キャンセル操作（キャンセルボタン、Esc 等）でダイアログを閉じる
- `currentAmount` は変更しない
- 入力はリセットする

## バリデーション

- 差分金額は整数
- `delta > 0` のときのみ確定可能
- `deltaAmountInput` が空、数値化できない、0、負数の場合は確定不可

実装方針：確定ボタンを `disabled` にして確定不可を表現する（エラーメッセージ表示は必須要件に含めない）。

## 表示仕様

- 現在金額は数値として表示する

## アクセシビリティ

- ダイアログ表示時、フォーカスがダイアログ内に移る
- Esc で閉じられる
