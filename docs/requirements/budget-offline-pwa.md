# 要件定義：PWA オフライン対応（Budget）

## 目的

Budget（`packages/client/budget`）を PWA として、ネットワークがない状況でも利用できるようにする。

## 対象範囲

- クライアント：`packages/client/budget`

## 機能要件

1. 初回にオンラインでアクセスした後は、オフラインでもアプリを起動できる
2. オフラインでもトップページ（`/`）が表示できる（アプリシェルが提供される）

## 前提・制約

- 初回アクセス（キャッシュ作成）にはネットワーク接続が必要
- ブラウザのストレージ/キャッシュを消去した場合、再度オンラインでの初期アクセスが必要

## 非機能要件

- 実装は PWA の一般的な仕組み（Service Worker によるキャッシュ）を用いる
- 開発体験を損なわないよう、開発時（`vite dev`）は通常 Service Worker を有効化しない
